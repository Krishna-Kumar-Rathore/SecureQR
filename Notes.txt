### **Step 4: ML Training (Optional)**
Only do this if you want to train custom models with your QR dataset.

Open a **third terminal** and run:

```bash
# Navigate to ml-training directory
cd ml-training

# Install Python dependencies (if not done already)
pip install -r requirements.txt

# Process your QR dataset (if you have the dataset)
python process_qr_dataset_windows.py

# Train ML models (after processing is complete)
python train_model.py
```

**Note:** ML training will only work if you have the QR code dataset in `dataset/QR Codes/`.

## **Quick Start Script (All at Once)**

Create a `start.sh` script in your project root:

```bash
#!/bin/bash

echo "ğŸš€ Starting SecureQR Project..."

# Start backend in background
echo "ğŸ“¡ Starting Backend..."
cd backend && npm run dev &
BACKEND_PID=$!

# Wait for backend to start
sleep 5

# Start frontend in background
echo "ğŸ¨ Starting Frontend..."
cd ../frontend && npm run dev &
FRONTEND_PID=$!

echo "âœ… Project started!"
echo "ğŸ“¡ Backend: http://localhost:5000"
echo "ğŸ¨ Frontend: http://localhost:5173"
echo ""
echo "Press Ctrl+C to stop all services"

# Wait for interrupt
trap "kill $BACKEND_PID $FRONTEND_PID" INT
wait
```

Then run:
```bash
chmod +x start.sh
./start.sh
```

## **Troubleshooting:**

**Backend won't start:**
- Ensure port 5000 is free
- Check Node.js version (18+)
- Verify .env file exists

**Frontend won't connect:**
- Ensure backend is running first
- Check for CORS errors in browser console
- Verify API URL in `frontend/src/utils/api.js`

**ML Training fails:**
- Ensure Python 3.8+ is installed
- Install dependencies: `pip install -r requirements.txt`
- Check if dataset exists in correct location
